# BioGrip SW 사양서

## 1. 개요 (Introduction)

### 1.1. 문서 목적 및 범위 (Purpose & Scope)

본 사양서는 Arduino Nano 33 BLE 임베디드 펌웨어 아키텍처 및 복합 생체신호(PPG, GSR) 처리 알고리즘의 소프트웨어 구현 사양을 규정한다. 제한된 임베디드 자원 환경에서 다중 주기의 센서 샘플링을 실시간으로 보장하고, Welford 알고리즘 및 이동 창 기법을 통해 스트레스 유효 특징점을 저지연으로 추출하여 BLE 5.0 가상 시리얼 프로토콜로 상향 전송하는 펌웨어 내부 로직의 명세 정립을 범위로 한정한다.

### 1.2. 소프트웨어 개발 및 구동 환경 (Development Environment)

* **배포 및 빌드 플랫폼:** Arduino IDE 인터페이스, Nordic nRF52840 기반 Mbed OS 가상화 런타임 레이어.
* **주요 종속 라이브러리 인터페이스 명세:**
* 
`<Wire.h>`: Fast Mode(400kHz) I2C 버스 제어 및 하드웨어 동기화.


* 
`"MAX30105.h"`: 레지스터 피킹 기반 광학 센서 제어 및 FIFO 버퍼 제어.


* 
`<ArduinoBLE.h>`: BLE 5.0 스택 가동, GATT 서비스 등록 및 Notify 전송 엔진 제어.

### 2.2. 태스크 및 타이밍 다이어그램 (Task Scheduling & Timing)

* 
**비동기 이벤트 스캔:** 메인 루프 인입 시 마다 `BLE.poll()` 함수를 호출하여 무선 이벤트 커맨드를 상시 스캔한다.


* 
**주기적 태스크 스케줄링:** 하드웨어 블로킹(delay)을 배제하고 `millis()` 타임스탬프 편차 비교문을 통해 멀티 타임슬롯을 구현한다.


* 
**PPG 계측 모듈:** 20ms 간격(`PPG_SAMPLE_INTERVAL_MS = 10`)으로 하드웨어 타이머를 분할하여 고빈도 맥파를 누수 없이 수집한다.


* 
**GSR 계측 모듈:** 100ms 간격(`GSR_SAMPLE_INTERVAL_MS = 100`) 분기를 통하여 피부전도도 신호를 샘플링하고 SCL 이동 창을 갱신한다.





### 2.3. 글로벌 시스템 데이터 구조 (Global System States & Variables)

시스템의 데이터 무결성을 보장하기 위해 핵심 제어 플래그 및 다중 센서 정규화 변수를 전역 공간에 할당하여 중앙 관리한다.

* 
`sys_measurementActive`: 시스템 계측 가동 및 내부 버퍼 연산 승인 여부를 제어하는 바이너리 플래그.


* 
`sys_integratedStressZ`: PPG 및 GSR의 유효 Z-Score 지표들을 상호 결합한 최종 복합 스트레스 지수.


* 
`sys_validFeatureCount`: 런타임 프레임에서 신호 품질 표준(SQI) 및 유효 범위를 충족한 활성 특징점의 개수.

## 3. 시스템 모드 및 시퀀스 제어 알고리즘 (System Mode & Sequence Control)

### 3.1. 시간 경과 기반 상태 전이 매트릭스 (State Transition Matrix)

시스템은 측정 개시 후 경과 시간(`elapsedMs`)을 기반으로 `getMode()` 함수를 호출하여 내부 물리 모드를 동적으로 전이하고 알고리즘 제어 규칙을 차별화한다.

```
[START] ➔ STABILIZING ➔ CALIBRATION ➔    REST     ➔ TRANSITION ➔   STRESS   ➔ [DONE]
          (0 ~ 15s)    (15 ~ 75s)   (75 ~ 375s)   (375 ~ 405s)  (405 ~ 675s)  (675s~)

```

### 3.2. 세부 모드별 소프트웨어 동작 사양

* 
**`STABILIZING` (0초 $\le$ $t$ $<$ 15초):** 센서 구동 직후 광학 필터 및 회로 전하 안정화를 유도하는 구간으로 특징점 연산을 제한한다.


* 
**`CALIBRATION` (15초 $\le$ $t$ $<$ 75초):** 누적 60초간 피험자 고유의 기저 생체신호 통계량을 실시간 빌드업하는 구간으로, Welford 모듈을 가동하여 개별 특징점의 평균과 분산 제곱합을 런타임 누적한다.


* 
**`REST` (75초 $\le$ $t$ $<$ 375초):** 편안한 상태에서의 생체 데이터를 30초 이동 창 기반으로 실시간 특징 추출 및 스트레스 평가를 정상 가동하는 구간이다.


* 
**`TRANSITION` (375초 $\le$ $t$ $<$ 405초):** 휴식기에서 스트레스 자극기로 전환되는 시점으로, 피험자의 과도한 거동에 의한 물리 노이즈를 차단하기 위해 30초간 가용 특징점 판단 및 Z-Score 연산을 일시 서스펜드하고 버퍼를 클리어한다.


* 
**`STRESS` (405초 $\le$ $t$ $<$ 675초):** 스트레스 부하 과제 수행 중의 생체 지표를 분석하여 실시간 스트레스 상태를 마이닝하는 액티브 판정 구간이다.

## 4. 센서 신호 처리 및 특징 추출 알고리즘 (Signal Processing & Feature Extraction)

### 4.1. PPG (광학) 신호 처리 모듈

* 
**피부 미접촉 예외 제어:** 기득한 IR 광량이 하한선 임계값(`ppg_NO_FINGER_IR_THRESHOLD = 10000`) 미만으로 떨어지는 카운트가 연속 10회(`ppg_NO_FINGER_CONFIRM_COUNT`) 충족 시 즉시 `loop()` 내 처리를 서스펜드하여 피부 이탈 시의 의사 피크 매칭 및 데이터 오염을 예방한다.


* **실시간 HRV 특징점 산출 기법:**
* 동적 임계 필터를 통과한 유효 피크 간의 시계열 차분을 통해 실시간 IBI(Inter-Beat Interval)를 추출하여 30개 크기의 정적 원형 큐 버퍼(`ppg_ibiBuf[30]`)에 격리 저장한다.


* 30초 단위 가용 프레임 내에서 전체 데이터를 재순회하지 않고, 지정된 정적 배열 크기 내에서 다이렉트 인덱싱 평균 및 표준편차 수식을 가동하여 실시간 분당 심박수(`ppg_meanHR`), 시간 영역 HRV 지표인 `ppg_sdnn` 및 `ppg_rmssd`를 독립 도출한다.





### 4.2. GSR (피부전기) 신호 처리 모듈

* 
**SCL(Tonic) 분리 필터 스펙:** 아날로그 변환 전압 데이터를 미디언 필터링한 후, 150개 배열 공간(`gsr_maBuf[150]`)을 활용한 가속형 이동평균(Moving Average) 필터를 가동하여 기저선 성분인 `sclBaseline`을 매 샘플링 마다 연속 연산한다.


* 
**메모리 최적화형 Welford 통계 추적기:** `CALIBRATION` 구간 동안 10Hz 주기로 밀려드는 600개 이상의 전도도 데이터 프레임에 대해 배열 저장 방식 대신, 재귀적 관계식을 이용한 싱글 패스(Single-pass) 방식으로 런타임 샘플 표준편차(`gsr_baselinePhasicStd`, `gsr_baselineSclStd`)를 도출함으로써 임베디드 SRAM 자원 고갈을 물리적으로 차단한다.

## 5. 다중 센서 융합 스트레스 분류 및 데이터 축소 기법 (Data Fusion & Reduction)

### 5.1. 실시간 Z-Score 정규화 알고리즘

* 피험자 고유 생체 변동성을 동적 캘리브레이션하기 위해, 추출된 실시간 지표들을 매 초 마다 정규화 스케일로 투영한다.


* 이때 자율신경계 균형에 의해 스트레스 상태에서 수치 하락을 보이는 HRV 지표 계열(`ppg_sdnn`, `ppg_rmssd`)은 정규화 함수 수식에 부호 반전(-) 연산을 결합 적용하여 통합 매트릭스의 방향성을 일치시킨다.



$$ppg\_sdnnStressZ = - \frac{ppg\_sdnn - ppg\_baselineSDNNMean}{ppg\_baselineSDNNStd}$$






### 5.2. 다중 센서 데이터 융합 알고리즘 (Sensor Fusion)

* 
**특징 유효성 동적 평가:** `sys_ppgReadyForBle` 및 `sys_gsrReadyForBle` 상태 제어 메커니즘을 통해 피부 접촉 상태, SQI 점수, 윈도우 신선도를 다각도로 교차 검증한다.


* 
**융합 판정 임계 조건:** 현재 런타임 초에서 최종 유효성이 입증된 개별 Z-Score 지표의 총 개수가 최소 요구 사양인 3개 이상(`sys_validFeatureCount >= 3`)을 완벽히 만족하는 프레임에 한해서만 산술 평균 연산을 최종 융합 가동한다. 도출된 `sys_integratedStressZ` 결과값이 융합 임계값인 $1.0$ 이상 도달 시 사용자의 현재 상태를 `Stress` 상태로, 미달 시 `Rest` 상태로 바이너리 라벨링을 수행한다.



### 5.3. 저전력 BLE 데이터 패킹 아키텍처 (Data Reduction)

* 
**비트 압축 아키텍처:** 고용량의 원시 생체 파형 파형 데이터를 실시간 전송하는 무선 대역폭 낭비를 원천 지양하고, 시스템 최종 단계에서 융합된 7개의 핵심 Feature 데이터만 바이트 정렬 구조체(`BleFeatureData`)로 구조화하여 50초 주기(`BLE_NOTIFY_INTERVAL_MS`)로 버스트 전송한다.


* **데이터 축소율 성과 반영:** 초당 50Hz의 4바이트 원시 데이터들을 무선으로 상시 전송하는 기존 방식과 비교하여, 본 비트 패킹 전송 메커니즘은 무선 가동 대역폭 및 저전력 데이터 송신 전송률을 대략 95% 이상 획기적으로 축소(Data Reduction)하여 임베디드 저전력 무선 통신 무결성을 실현한다.

## 6. 인터페이스 및 통신 프로토콜 (Interfaces & Protocols)

### 6.1. 무선 BLE 서비스 및 특성(GATT 테이블) 사양

* 
**GATT 서비스 레이아웃 정의:** 서비스 고유 UUID(`19B10000-E8F2-537E-4F6C-D104768A1214`) 하위에 제어 및 융합 데이터 전송용 특성을 배치한다.


* **`featureDataChar` (UUID: `19B10001-...`):** `BLERead | [cite_start]BLENotify` 속성으로 정의되며, `#pragma pack(1)` 컴파일러 지시어를 통해 구조체 멤버 간 패딩 바이트를 강제 거부한 32바이트 압축 스트림 페이로드를 직렬화하여 호스트 디바이스로 밀어낸다.



### 6.2. 디버깅 및 시리얼 통신 커맨드 스펙

* 
**런타임 호스트 명령어 매핑:** 시리얼 UART 포트를 개방하여 외부 호스트로부터 단자 명령어를 접수 처리한다.


* 문자 `'S'` / `'s'` 인입 시: `startMeasurement()` 가동, 내부 타이머 동기화 및 링 버퍼 올 클리어 제어.


* 문자 `'X'` / `'x'` 인입 시: `sys_measurementActive = false` 변환 및 센서 제어 루프 연산 일시정지.




* 
**CSV 포맷 출력 엔진:** 개발 및 실험 데이터 정밀 추출을 위해 `ENABLE_SERIAL_CSV = true` 플래그 매칭 시, 타임스탬프와 모드 정보, 필터링 중간 특징 지표, 개별 Z-Score 및 최종 스트레스 판정 상태 명사를 쉼표(`,`) 기호로 엄격히 구분한 단일 라인 CSV 포맷 스트리밍 아키텍처를 제공한다.
