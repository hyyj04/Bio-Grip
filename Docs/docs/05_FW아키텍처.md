# BioGrip FW 아키텍처

## 1. 개요 (Introduction)

### 1.1. 펌웨어 아키텍처 설계 목적 (Purpose)

본 문서의 목적은 자원이 제한된 임베디드 하드웨어 환경 내에서 고주파 맥파 신호와 저주파 피부전도도 신호의 실시간성(Real-time)을 완전 보장하고, 메모리 단편화를 방지하기 위한 소프트웨어 계층 구조 및 데이터 흐름을 정의하는 데 있다. 런타임 제어 메커니즘과 데이터 격리 사양을 규정함으로써 시스템의 구조적 무결성과 연산 신뢰성을 증명한다.

### 1.2. 시스템 하드웨어 타깃 환경 (Target Hardware Environment)

* **하드웨어 칩셋:** nRF52840 (ARM Cortex-M4 32-bit RISC with FPU) 탑재 Arduino Nano 33 BLE 모듈.
* **커널 환경:** Mbed OS 가상화 런타임 레이어를 통해 하드웨어 자원이 추상화되며, 펌웨어는 상위 단일 스레드 기반의 비동기 이벤트 폴링 및 시간 분할 아키텍처로 구동된다.

## 2. 펌웨어 계층 구조 (Firmware Layered Architecture)

### 2.1. 전체 상위 계층 아키텍처 (High-Level Layered Diagram)

펌웨어는 모듈 간의 결합도를 낮추고 이식성을 높이기 위해 엄격한 5개 다층 레이어 구조로 분리 설계된다.

```
+---------------------------------------------------------------------------------+
| 5. 통신 및 출력 레이어 (Communication / Output Layer)                           |
|    - BLE GATT 서비스 특성 제어 | Serial UART CSV 스트리밍 엔진                  |
+---------------------------------------------------------------------------------+
                                        ▲
+---------------------------------------------------------------------------------+
| 4. 애플리케이션 및 비즈니스 로직 레이어 (Application / Business Logic Layer)       |
|    - 상태 전이 엔진(getMode) | 복합 스트레스 다중 센서 융합 판단 모듈            |
+---------------------------------------------------------------------------------+
                                        ▲
+---------------------------------------------------------------------------------+
| 3. 데이터 처리 및 신호 필터 레이어 (Data Processing / Signal Filtering Layer)    |
|    - PPG Pseudo-HPF | GSR 미디언 필터 및 MA | Welford 실시간 통계 계산 모듈      |
+---------------------------------------------------------------------------------+
                                        ▲
+---------------------------------------------------------------------------------+
| 2. 하드웨어 추상화 레이어 (Hardware Abstraction Layer - HAL)                    |
|    - Wire API를 통한 I2C 버스 제어 | analogRead 기반 ADC 주변장치 인터페이스      |
+---------------------------------------------------------------------------------+
                                        ▲
+---------------------------------------------------------------------------------+
| 1. 물리 하드웨어 및 센서 레이어 (Physical Hardware & Sensor Layer)               |
|    - nRF52840 MCU | MAX30105 광학 센서 레지스터 | GSR 분압 및 보호 회로 단자       |
+---------------------------------------------------------------------------------+

```

### 2.2. 하드웨어 추상화 및 드라이버 레이어 (HAL & Driver Layer)

* 
**I2C 버스 관리 인터페이스:** `<Wire.h>` 드라이버를 통해 고속 모드(`I2C_SPEED_FAST`, 400kHz) 버스를 개방하고 , `"MAX30105.h"` 하드웨어 API와 연동하여 칩 내부 FIFO 버퍼 및 LED 발광 레지스터 인터페이스를 독점 통제한다.


* 
**아날로그 단자 계측 인터페이스:** 기성 마우스의 물리적 전압 입력을 위해 `analogRead(GSR_PIN)` 인프라 채널을 링크하며 , 내부 10비트 SAR ADC 레지스터와 직접 결합 제어한다.



### 2.3. 미들웨어 및 미들 필터 레이어 (Signal Filtering Layer)

* 
**메모리 재사용형 정적 버퍼 구조:** 신호 정제 및 특징점 연산을 위해 가동되는 모든 윈도우는 동적 할당을 영구 배제한다.


* 
**GSR 미디언 필터:** 3개 크기의 정적 환형 구조인 `gsr_medBuf[3]`를 사용하여 물리 스파이크 노이즈를 런타임 거부한다.


* 
**SCL 이동평균 필터:** 150프레임 고정 크기의 정적 링 버퍼 `gsr_maBuf[150]`를 매핑하여 $O(1)$ 복잡도의 이동합 연산을 실현한다.





### 2.4. 애플리케이션 및 비즈니스 로직 레이어 (Application Logic Layer)

* 
**런타임 제어 엔진:** 실험 경과 시간에 따라 시스템 컨텍스트 모드를 변경하는 `getMode()` 제어 엔진이 탑재된다.


* 
**실시간 통계 계산 모듈:** 싱글 패스 분산 추적 함수인 `sys_updateWelford()`를 배치하여 , 누적 샘플 데이터에 대한 기저 편차 매트릭스를 실시간 갱신 빌드업한다.

## 3. 태스크 관리 및 스케줄링 아키텍처 (Task Management & Scheduling)

### 3.1. 슈퍼루프(Super-Loop) 기반 비동기 의사-멀티태스킹 (Pseudo-Multitasking)

임베디드 소프트웨어의 무결성을 확보하기 위해 CPU 자원을 장시간 점유하는 하드웨어 블로킹 함수(`delay()`)를 소프트웨어 전체에서 완전히 배제한다. 메인 `loop()` 엔진 내부에서 `millis()` 타임스탬프 편차 비교 구문(Non-blocking 타이머 플래그 기석)만을 활용하여 다중 주기의 의사-멀티태스킹 스케줄링 아키텍처를 독립 수립한다.

### 3.2. 실시간 샘플링 타임 슬롯 배치 (Time-Slot Allocation)

시간 분할 방식에 의거하여 다음과 같이 명확히 격리된 4개의 타임슬롯 프로세스를 스케줄링 제어한다.

```
[loop() 진입]
   │
   [cite_start]├─── (now - ppg_lastSampleTime >= 10ms) ───► [PPG 태스크 슬롯] (50Hz) [cite: 46, 166]
   [cite_start]│                                              └─ IR 취득, EMA HPF 필터, 피크 마이닝, IBI 계산 [cite: 101, 102, 103, 104, 106, 107]
   │
   [cite_start]├─── (now - gsr_lastSampleTime >= 100ms) ──► [GSR 태스크 슬롯] (10Hz) [cite: 30, 165]
   [cite_start]│                                              └─ ADC 변환, 미디언 정제, SCL MA, Phasic 피크 검출 [cite: 69, 71, 77, 87, 90]
   │
   [cite_start]├─── (now - lastPrintTime >= 1000ms) ──────► [융합 및 출력 슬롯] (1Hz) [cite: 167, 168]

```

## 4. 데이터 플로우 및 인터-태스크 통신 (Data Flow & Variable Isolation)

### 4.1. 전역 시스템 컨텍스트 관리 (Global System Context)

* 
**시스템 초기화 릴리즈:** 호스트 제어 명령 수신 시 `sys_resetAll()` 아키텍처를 기동하여 통계 누적 카운터와 유효성 플래그 변수들을 초기화 상태로 올 클리어 제어한다.


* 
**예외 처리 동적 플래그:** 피부 미접촉 또는 전극 탈착 감지 시, `noFinger` 및 `gsr_contactOk` 전역 상태 변수가 즉시 반영되어 서브 모듈의 오진 연산을 즉각 차단한다.



### 4.2. 센서 모듈 간 독립성 및 상호 참조 아키텍처

데이터 가공 흐름은 계층적 전방향 아키텍처를 준수하여 모듈 간 파이프라인의 간섭을 최소화한다.

* 
**PPG 단독 흐름:** `ppg_latestIrRaw` $\rightarrow$ `ppg_latestFiltered` $\rightarrow$ `ppg_rawIbiMs` $\rightarrow$ `ppg_ibiBuf` $\rightarrow$ HRV 지표 추출.


* 
**GSR 단독 흐름:** `gsr_latestRawAdc` $\rightarrow$ `medianGsr` $\rightarrow$ `sclBaseline` / `phasic` $\rightarrow$ SCR 피크 카운팅.


* **최종 데이터 융합 수렴 구조:**
독립 격리 연산되어 출력된 센서별 개별 특징 Z-Score 값들은 오직 최상위 애플리케이션 레이어의 `sys_updateCombinedStateAndPrint()` 단일 수렴 함수 내부에서만 상호 통합 참조되며, 변수 오염 및 종속성 뒤엉킴 현상을 차단한다.

## 5. 무선 통신 및 메모리 최적화 아키텍처 (Communication & Memory Architecture)

### 5.1. BLE 5.0 GATT 프로토콜 스택 사양

* 
**서비스 및 특성 인스턴스 설계:** `<ArduinoBLE.h>` 라이브러리를 가동하여 커널 공간 내에 `bioDataService` GATT 테이블을 인스턴싱하고 , 독출/통지용 `featureDataChar`와 제어용 `controlPointChar` 핸들러를 물리 주소 공간에 바인딩한다.


* 
**Write 이벤트 비동기 캡처:** 외부 원격 제어 디바이스의 데이터 쓰기 동작 발생 시 `controlPointChar.written()` 인터럽트 폴링 로직이 비동기 가동되어 `startMeasurement()` 시퀀스를 동적으로 실행하도록 구조화되어 있다.



### 5.2. 메모리 정렬을 통한 패킷 최적화 (Data Packing)

* 
**구조체 바이트 패킹 아키텍처:** 32비트 임베디드 컴파일러가 자원 최적화를 위해 멤버 변수 사이에 임의로 삽입하는 가상 패딩 바이트(Padding Byte)를 배제하기 위해 컴파일러 지시어 `#pragma pack(1)` 레이아웃 아키텍처를 강제 정의한다.


* 
**무선 대역폭 절감 극대화:** 4바이트 크기의 단일 정밀도 부동 소수점(`float`) 변수들과 1바이트 코드 변수들이 빈 곳 없이 물리 바이트 메모리에 연속 밀집 바인딩되도록 통제한다. 원시 생체 유동 파형 전송 방식 대비 본 구조체 패킹Notify 엔진 아키텍처는 무선 가동 주파수 대역 및 물리 송신 가용 대역폭을 95% 이상 대폭 저감(Data Reduction)시켜 통신 신뢰성을 완벽히 확보한다.



### 5.3. 휘발성 메모리(SRAM) 가용성 및 정적 할당 최적화

* **동적 메모리 할당(`malloc`/`new`) 원천 차단:** 장시간 마우스를 운용하는 다중 프레임 실험 환경에서 메모리 해제 지연에 따른 SRAM 단편화 및 릭(Leak) 현상으로 시스템이 다운되는 오류를 방지하기 위해, 런타임 내의 모든 메모리 맵은 빌드 타임 시 정적 할당 영역(`SRAM Static Area`)에만 배치되도록 강제 규정한다.
* 
**배열 크기 유한 한정 기술:** 실시간 이동 창 가동용 버퍼 배열 크기를 물리 하드웨어 수용 한계점 이내인 `gsr_maBuf[150]`(150바이트) , `ppg_ibiBuf[30]`(120바이트) 등으로 타이트하게 사전 규격화하여 구동 신뢰성을 극대화한다.
