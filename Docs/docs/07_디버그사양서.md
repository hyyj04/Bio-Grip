# BioGrip 디버그 사양서

## 1. 개요 (Introduction)

### 1.1. 디버그 사양서 목적 (Purpose)

본 사양서는 마우스 일체형 생체신호 계측 하드웨어와 실시간 스트레스 분류 알고리즘의 동작 무결성을 검증하기 위한 디버깅 인터페이스 및 시험 규격을 정의한다. 물리적인 동잡음과 예외 상황(피부 탈착, 무효 데이터 인입) 속에서 계측 시스템의 정밀도와 신뢰성을 추적·보정하고, 호스트 분석 소프트웨어(Python 등)와의 연동을 위한 데이터 추출 과정을 체계화하는 데 목적이 있다.

### 1.2. 디버그 대상 인터페이스 (Target Interfaces)

* **하드웨어 디버그 포트:** 칩셋 내장 USB 가상 COM 포트를 이용한 UART 시리얼 통신 인터페이스 (통신 속도: $115,200\text{ bps}$ 고정).
* **소프트웨어 무선 디버그 포트:** BLE 5.0 GATT 프로토콜 레이어. 제어 핸들러 특성(`controlPointChar`) 및 데이터 버스트 패킷 통지 특성(`featureDataChar`) 프로토콜 스택 모니터링.

## 2. 시리얼(UART) 디버깅 및 데이터 로깅 사양 (UART Logging Specification)

### 2.1. 실시간 데이터 스트리밍 사양 (`ENABLE_SERIAL_CSV`)

* 
**런타임 컴파일 스위치 제어:** 펌웨어 최상단의 `ENABLE_SERIAL_CSV = true` 가동 시, 시스템 매 초마다 시리얼 버스를 통해 규격화된 로깅 스트림을 방출한다.


* 
**최초 1회 헤더 방출 조건:** 시스템 가동 및 버퍼 리셋 직후, `sys_csvHeaderPrinted = false` 플래그 조건을 인지하여 호스트 수신기가 파싱할 수 있는 아래 규격의 CSV 단일 라인 텍스트 헤더를 최초 1회 무조건 출력해야 한다.


```
timestamp,mode,raw_gsr,ir_raw,filtered_ppg,ppg_mean_hr,ppg_sdnn,ppg_rmssd,gsr_scl,gsr_scr_amp,gsr_scr_freq,ppg_sdnn_z,ppg_rmssd_z,gsr_scl_z,gsr_scr_amp_z,gsr_scr_freq_z,final_ppg_stress_z,final_gsr_stress_z,valid_feature_count,integrated_stress_z,integrated_state

```



### 2.2. 로깅 필드 매핑 및 데이터 분석 규격

매 주기로 출력되는 시리얼 패킷 데이터 필드는 신호 처리 단계에 따라 다음과 같이 3개 레이어로 엄격히 분류 파싱되어 검증 가독성을 확보한다.

* 
**센서 원시 데이터 레이어:** 경과 시간(`elapsedMs`), 현재 물리 실험 구간 모드(`mode`), GSR ADC 원시값(`gsr_latestRawAdc`), PPG 적외선 광량 원시값(`ppg_latestIrRaw`), DC 제거 정제 파형(`ppg_latestFiltered`) 검증.


* 
**알고리즘 중간 특징 지표 레이어:** 30초 이동 윈도우 내에서 실시간 누적 추출된 평균 심박수(`ppg_meanHR`), SDNN(`ppg_sdnn`), RMSSD(`ppg_rmssd`), 피부 전도도 수준 평균(`gsr_latestSclMean30s`), SCR 평균 진폭(`gsr_latestScrAmplitudeMean30s`), 분당 SCR 발생 빈도(`gsr_latestNsScrFrequencyPerMin`) 모니터링.


* 
**정규화 및 최종 융합 레이어:** 베이스라인 대비 개별 특징값별 실시간 Z-Score 변환값 , 최종 센서 모듈별 수렴 Z-Score , 융합 조건을 만족한 활성 특징점 개수(`sys_validFeatureCount`) , 최종 결합 스트레스 지수(`sys_integratedStressZ`) 및 문자열 판정 상태(`sys_integratedStressState`)의 무결성 실시간 확인.

## 3. 시리얼 디버그 커맨드 시스템 (Serial Command System)

### 3.1. 런타임 제어 명령어 스펙 (Runtime Host-to-Device Commands)

하드웨어 계측 프로세스를 강제 통제하기 위해 호스트 PC 터미널 인터페이스로부터 1바이트 문자 제어 커맨드를 접수 처리하는 디버그 파이프라인을 수립한다.

* 
**'S' 또는 's' (Start) 명령어:** 플래그 변수 `sys_measurementActive = true` 전이를 트리거하며, 내부 함수 `sys_resetAll()`을 강제 호출하여 링 버퍼 인덱스 올 클리어, 통계 통계 추적 매트릭스 리셋 및 기준 타이머 `ppg_startTimeMs`, `gsr_startTimeMs`를 동기화하여 측정 세션을 즉시 개시한다.


* 
**'X' 또는 'x' (Halt) 명령어:** 제어 플래그 `sys_measurementActive = false` 상태로 동적 역전시켜, 가동 중인 센서 샘플링 타이머 연산 및 특징점 마이닝 로직을 즉시 정지(Suspend)시킨 후 하드웨어 대기 모드로 진입시킨다.



### 3.2. 상태 전이 인터커넥션 디버깅

외부 호스트 명령어 강제 인입에 따른 전역 시스템 컨텍스트 변수의 즉각적인 비동기 매핑 무결성을 시리얼 터미널 반향 출력을 통해 상호 검증한다.

## 4. 무선(BLE) 프로토콜 디버깅 사양 (BLE Protocol Debugging)

### 4.1. BLE 제어 포인트 (`controlPointChar`) 인터럽트 디버깅

원격 무선 기기로부터 가용 제어 바이트가 기입(`controlPointChar.written()`)되는 비동기 이벤트를 디버깅 캡처한다.

* 16진수 값 `0x01` 수신 시: 시리얼 커맨드 'S'와 연동되는 `startMeasurement()` 시퀀스 내부 콜백 정상 기동 여부 검증.


* 16진수 값 `0x02` 수신 시: 계측 액티브 플래그를 소거하여 전체 무선 전송 시퀀스를 정상적으로 홀드(Hold) 처리하는지 검증.



### 4.2. 페이로드 구조체 메모리 정렬 및 데이터 패킹 검증

* 
**메모리 패딩 누수 검증:** 컴파일러 지시어 `#pragma pack(1)` 레이아웃에 의해 빌드된 32바이트 구조체 `BleFeatureData`가 하드웨어 아키텍처 메모리단에서 패딩 빈 공간 없이 완벽히 정렬 결합되었는지 바이트 스트림 직렬화 모니터링을 통해 화이트박스 검증을 수행한다.


* 
**결측치(NAN) 예외 에러 코드 매핑 규칙 테스트:** 각 센서 특징점 추출기 및 Z-Score 융합 모듈 연산 중, 품질 역치 미달 등의 원인으로 부동 소수점 결측치(`isnan`) 오류가 내부적으로 유발될 경우, 통신 프로토콜 규격에 의거하여 무선 송신 전용 특성 페이로드 필드에 정상 스펙 대체값인 `-1.0` 상수를 오염 없이 정상 치환 패킹하여 무선 전송하는지 집중 테스트한다.



### 4.3. 무선 데이터 패킷 Notify 간격 타이밍 디버깅

* 
**버스트 전송 주기 검증:** 50,000ms 주기(`BLE_NOTIFY_INTERVAL_MS = 50000UL`) 플래그 타이머 조건문 진입 시 , 통지 함수 `featureDataChar.writeValue()`가 Mbed OS BLE 스택 내 전송 큐에 정상 커밋되는지 무선 프로토콜 패킷 스니퍼 장비를 통해 패킷 드롭 및 전송 지연 레이턴시를 정밀 계측 검증한다.

## 5. 알고리즘 단계별 예외 처리 및 화이트박스 테스트 (Algorithm Test & Validation)

### 5.1. 피부 미접촉(No-Finger) 판단 예외 디버깅

* 
**센서 탈착 예외 천이 루틴 시험:** 엄지손가락 이탈 모사 환경을 구성하여 광학 센서 로우 데이터 값을 임계 역치(`ppg_NO_FINGER_IR_THRESHOLD = 10000`) 미만 상태로 고의 유도한다.


* 
**카운터 포화 가동 검증:** 프레임 누적 변수 `ppg_consecutiveNoFinger` 가 동적 포화 조건인 10회(`ppg_NO_FINGER_CONFIRM_COUNT`)에 도달하는 시점에 , `getMode()` 함수 내부에서 상태 문자열 `"NO_FINGER"`를 에러 명사로 정상 리턴하고 후속 알고리즘 파이프라인 연산을 동적 우회 처리하는지 화이트박스 단독 유효성 검증을 수행한다.



### 5.2. GSR 탈착 및 데이터 오염 예외 디버깅

* **전극 오픈 회로(Open Circuit) 테스트:** 마우스 조작 중 사용자가 손을 완전히 떼어 GSR 입력 라인이 개방되는 가혹 조건을 유도한다.
* 
**하한선 트리거 작동 검증:** 로우 ADC 데이터가 계측 하한 역치(`gsr_MIN_VALID_GSR_ADC = 10`) 미만으로 떨어지는 즉시 전역 접촉 플래그 `gsr_contactOk = 0`으로 디매핑 처리가 수행되는지 확인한다. 이에 연동되어 회복 카운터 `gsr_contactRecoveryCount`가 10프레임으로 셋업되며 시스템 융합 엔진 내부에서 GSR 모듈 가용성 변수 `sys_gsrReadyForBle = false`로 디세이블 차단 처리되는지 유기적 조건문 동작의 무결성을 시험한다.



### 5.3. 베이스라인(Calibration) 빌드업 통계 무결성 디버깅

* 
**Welford 알고리즘 수렴 시험:** 초기 시스템 기동 후 `CALIBRATION` 구간(누적 시간 15초 이상 ~ 75초 미만) 동안 수집되는 통계 데이터 갱신 구조를 추적한다. 60초 만료 시점(`COMMON_BASELINE_END_MS`)에 도달했을 때 , 수 수백 프레임의 연산 데이터 분산 제곱합 변수 `gsr_baselinePhasicM2` , `gsr_baselineSclM2` 가 임베디드 오버플로우 없이 정밀한 최종 표준편차 수치(`gsr_baselinePhasicStd`, `gsr_baselineSclStd`)로 정상 수렴 수득되어 전역 변수 `gsr_baselineReady = true` 플래그를 수립하는지 화이트박스 한계점 테스트를 시행한다.


* 
**통합 판단 임계 조건 테스트:** 다중 지표 결합 시, 활성 특징점 개수 카운팅 수식이 설계 규격 최소치 기준인 3개 이상(`sys_validFeatureCount >= 3`) 조건을 충족하는지 강제 인입 테스트를 진행한다. 기준 충족 시에만 최종 통합 가중 산술 평균 연산문이 개시되어 바이너리 상태가 정상 출력되는지 조건문 분기 무결성을 최종 실증 검증한다.
