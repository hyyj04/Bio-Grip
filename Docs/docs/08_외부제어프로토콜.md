# BioGrip 외부 제어 프로토콜

## 1. 개요 (Introduction)

### 1.1. 프로토콜 명세 목적 (Purpose)

본 사양서는 BioGrip 스마트 마우스 하드웨어와 외부 호스트 제어 단말(PC 분석 시스템, 모바일 모니터링 대시보드 앱 등) 간의 통신 인터페이스 규격을 표준화하는 데 목적이 있다. 외부 시스템이 하드웨어를 원격 통제하고, 마우스로부터 전송되는 압축 특징점 패킷 및 시리얼 데이터 스트림을 에러 없이 정밀하게 파싱·연동할 수 있도록 데이터 포맷과 교환 규칙을 명세한다.

### 1.2. 통신 환경 및 계층 구조 (Communication Layer)

* **무선 인터페이스:** `Bluetooth Low Energy (BLE) 5.0` 기반의 GATT(Generic Attribute Profile) 아키텍처를 채택하여 실시간 상태 알림(Notify) 중심의 저전력 버스트 전송을 수행한다.
* **유선 인터페이스:** USB 가상 COM 포트를 이용한 `UART 시리얼` 통신을 기반으로 하며, 비동기식 문자열 커맨드 수신 및 CSV 포맷 데이터 로깅 스트리밍을 지원한다.

## 2. BLE (Bluetooth Low Energy) 통신 프로토콜 명세

### 2.1. BLE GATT 레이아웃 및 프로토콜 서비스 정의

* 호스트 디바이스가 BioGrip 하드웨어를 무선 탐색 및 식별할 수 있도록 커널 내에 독립적인 전용 GATT 서비스를 빌드한다.


* 
**서비스 고유 UUID 사양:** `19B10000-E8F2-537E-4F6C-D104768A1214` 



### 2.2. 제어 포인트 특성 (`controlPointChar`) 쓰기(Write) 프로토콜 사양

* 
**특성 UUID 명세:** `19B10002-E8F2-537E-4F6C-D104768A1214` 


* 
**통신 속성 사양:** `BLEWrite` 


* **호스트-디바이스(Host-to-Device) 원격 제어 명령 코드 프로토콜:**
호스트 단말이 본 특성에 1바이트 명령 코드를 Write할 경우, 하드웨어는 다음과 같이 즉각적으로 시스템 컨텍스트 변수를 동적 매핑 제어해야 한다.



| 기입 명령 코드 (Hex) | 연동 제어 함수 및 플래그 변환 | 외부 제어 동작 사양 정의 |
| --- | --- | --- |
| **`0x01`** | <br>`startMeasurement()` 

 | 시스템 계측 세션 시작. 내부 링 버퍼 및 Welford 통계 처리 추적기 올 클리어 후 연산 개시.

 |
| **`0x02`** | <br>`sys_measurementActive = false;` 

 | 시스템 계측 세션 종료. 센서 샘플링 및 스트레스 판정 연산 루프 일시정지(Suspend).

 |

### 2.3. 생체 특징 데이터 특성 (`featureDataChar`) 통지(Notify) 프로토콜 사양

* 
**특성 UUID 명세:** `19B10001-E8F2-537E-4F6C-D104768A1214` 


* **통신 속성 사양:** `BLERead | [cite_start]BLENotify` 


* **구조체 바이트 패킹 및 정렬 사양:**
32비트 임베디드 컴파일러의 최적화 레이아웃에 의한 가상 패딩 바이트(Padding Byte) 개입을 원천 차단하기 위해 컴파일러 지시어 `#pragma pack(1)` 구조체를 적용한다. 수신단은 아래 명세된 비트 레벨 오프셋 단위로 바이트 스트림을 다이렉트 복사(De-serialization) 파싱해야 한다.


* **32바이트 고정 데이터 페이로드 레이아웃 명세:**

| 바이트 오프셋 | 필드 변수명 | 데이터 타입 | 스케일 / 에러코드 정의 및 세부 파싱 스펙 |
| --- | --- | --- | --- |
| **0 ~ 1** | `seq` | `uint16_t` | 패킷 송신 일련번호 (0 ~ 65535 환형 카운터).

 |
| **2 ~ 5** | `mean_hr` | `float` | 실시간 분당 심박수 원값. 결측치 수렴 시 **`-1.0`** 송신.

 |
| **6 ~ 9** | `final_ppg_stress_z` | `float` | 최종 PPG(HRV 지표 결합) 스트레스 Z-Score. 결측 시 **`-1.0`**.

 |
| **10 ~ 13** | `final_gsr_stress_z` | `float` | 최종 GSR(SCL/SCR 결합) 스트레스 Z-Score. 결측 시 **`-1.0`**.

 |
| **14 ~ 17** | `integrated_stress_z` | `float` | 다중 센서 융합 최종 통합 스트레스 Z-Score. 결측 시 **`-1.0`**.

 |
| **18** | `integrated_level` | `uint8_t` | 통합 이진 판정 코드 (`0`: 미정, `1`: Rest, `2`: Stress).

 |
| **19** | `mode_code` | `uint8_t` | 현재 런타임 물리 모드 코드 (`0`~`5` 범위 매핑 수득).

 |
| **20** | `flags` | `uint8_t` | 하드웨어 센서 무결성 및 시스템 제어 비트 플래그 (하단 명세 참고).

 |
| **21 ~ 31** | `reserved` | `uint8_t[11]` | 차후 지표 가중치 및 확장 변수 이식을 위한 예비 공간 패딩. |

## 3. UART Serial 시리얼 제어 프로토콜 명세

### 3.1. 시리얼 버스 기본 통신 사양

* 
**Baud Rate:** $115,200\text{ bps}$ 


* **Data Bits:** 8 bits
* **Parity Bit:** None
* **Stop Bit:** 1 bit
* **Flow Control:** None

### 3.2. 시리얼 호스트 제어 명령어 수신 스펙

호스트 PC 터미널에서 하드웨어로 USB 데이터 라인을 통해 아래의 1바이트 ASCII 문자를 전송할 경우, 임베디드 펌웨어는 수신 인터럽트 루프 즉시 해당 제어 시퀀스를 매핑 분기해야 한다.

* **`'S'` 또는 `'s'` (ASCII `0x53` / `0x73`):** BLE 명령 `0x01`과 물리적으로 동일하게 결합 구동됨. `startMeasurement()` 함수를 개시하여 전역 버퍼 및 카운터를 올 클리어한 후 측정을 개시한다.


* **`'X'` 또는 `'x'` (ASCII `0x58` / `0x78`):** BLE 명령 `0x02`와 동치 제어됨. 전역 가동 제어 변수 `sys_measurementActive = false` 상태로 동적 전환하여 센서 신호 처리 가동을 중지한다.

## 4. 실시간 데이터 파싱 및 로깅 프로토콜 사양 (Data Streaming)

### 4.1. CSV 스트리밍 프레임 규격 (`ENABLE_SERIAL_CSV`)

* 
**데이터 격리 토큰 규칙:** 각 데이터 필드는 ASCII 문자열로 구성되며, 각 변수 간의 분리는 쉼표(`,`) 기호로 엄격히 구분 스트리밍된다. 라인의 끝은 개행 문자(`\r\n`)로 식별한다.


* 
**최초 헤더라인 방출 프로토콜:** 시스템 측정 가동 직후 `sys_csvHeaderPrinted = false` 상태 조건문 인지 시, 호스트 파서의 필드 동기화를 위해 정확히 21개 명사로 이루어진 텍스트 플레인 헤더 라인을 단 1회 시리얼 버스로 방출해야 한다.

## 5. 예외 상황 통신 프로토콜 규격 (Exception & Status Flags)

### 5.1. 센서 가용성 비트 플래그 (`flags`) 필드 비트 마스크 사양

구조체 오프셋 20번에 실리는 1바이트 크기의 `flags` 필드는 마우스의 하드웨어 접촉 결함 및 데이터 무결성 유효 상태를 실시간 보증하는 핵심 비트마스크 필드이다. 외부 수신 시스템은 본 바이트에 논리곱(`AND`) 연산을 가동하여 시스템 상태 경고 대시보드를 동적으로 갱신해야 한다.

* **`Bit 0 (0x01)` : `sys_ppgReadyForBle` 무결성 비트**
* 
`1`이면 엄지손가락 접촉 상태가 청결하며 SQI 품질 필터를 통과한 HRV 특징점이 정상 산출 및 융합에 반영 중임을 보증함. `0`이면 맥파 연산 신뢰성 보류 상태를 명시.




* **`Bit 1 (0x02)` : `sys_gsrReadyForBle` 무결성 비트**
* 
`1`이면 손바닥/우측면 GSR 전극 밀착 상태가 정상이며 윈도우 신선도가 유효하여 통계 연산에 정상 결합 중임을 보증함. `0`이면 피부 전극 이탈(Open Circuit) 상태를 경고.




* **`Bit 2 (0x04)` : `labelMode` 구간 유효성 비트**
* 
`1`이면 현재 시간 축이 기준 캘리브레이션을 무결하게 마친 후의 순수 실제 데이터 수집 구역인 `REST` 또는 `STRESS` 구간에 진입해 있음을 명시함. `0`이면 시스템 전처리 준비 상태를 뜻함.




* **`Bit 3 (0x08)` : `sys_measurementActive` 시스템 활성 비트**
* 
`1`이면 마우스가 제어 명령을 정상 접수하여 내부 샘플링 루프를 액티브하게 런타임 구동 중임을 명시함. `0`이면 유휴 대기 상태임을 명시.





### 5.2. 예외 상태 코드 매핑 사양

피험자가 마우스 조작 중 손을 완전히 임의 이탈시키는 탈착 상황 유발 시, 외부 호스트 시스템과의 프로토콜 무결성을 위해 다음과 같은 특수 동적 코드를 상향 송신하도록 사양을 정의한다.

* 
**미접촉 상태 코드 전이:** `ppg_consecutiveNoFinger >= 10` 카운터 포화 만족 즉시 , 상향 BLE 패킷의 `mode_code` 필드 데이터를 특수 정의된 역치 코드인 `5`로 강제 치환 매핑하여 무선 통지한다. 수신 시스템은 해당 모드 코드를 캡처하는 즉시 화면에 **"NO_FINGER (피부 미접촉)"** 예외 안내 로그를 표출 제어해야 한다.
